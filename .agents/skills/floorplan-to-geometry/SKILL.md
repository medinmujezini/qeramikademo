# Floorplan → Geometry Pipeline

## Core principle
**AI is a raster preprocessor only.** Geometry is 100% deterministic from the cleaned raster. Re-running stages 04–05 with the same cleaned image MUST produce identical walls.

## Stages

```text
[01 Upload]    raster image → ImageBitmap
[02 Calibrate] click A, click B, enter meters → pixelsPerMeter (deterministic)
[03 AI Clean]  raster → 1-bit raster (walls=black, rest=white)        ← NON-deterministic
[04 Trace]     Potrace outline + orthogonalize → closed SVG paths     ← deterministic
[05 Walls]     paths → THREE.Shape → ExtrudeGeometry                  ← deterministic
```

## Data contracts

- **03 → 04**: ImageData (RGBA) or Uint8Array (1-bit), same WxH as upload, pure binary (wall=0, void=255).
- **04 → 05**: `Array<{ d: string, points: {x,y}[], bboxArea: number, id: string, enabled: boolean }>` — `d` is a closed SVG path, orthogonalized.

## Stage 03 — AI clean
**Remove** (fill white): text labels, dimension/north/scale annotations, door swing arcs + door leaf (keep the gap), all furniture/fixtures.
**Fill black**: window openings (parallel-line / hatched symbols). After fill, the window must read as SOLID WALL.
**Preserve**: wall lines at original thickness/position. Preserve T- and L-junctions exactly.

Implementation options, in order of preference:
1. **segmentation_inpaint** — SAM or fine-tuned floorplan model → class-specific fill. Most reliable.
2. **vlm_with_masks** — Vision LLM (e.g. Gemini Flash) returns bounding boxes for regions; client renders the cleaned binary raster from masks. Fastest to ship.
3. **img2img_diffusion** — only with structural ControlNet edge-lock and seed-deterministic. High hallucination risk.

**Recommended split**: edge function asks AI for mask regions only (JSON), client binarizes + applies masks. Keeps the AI contract clean and swappable.

Failure modes to watch:
- Hallucinated walls
- "Cleaning" load-bearing wall hatching mistaken for furniture
- Closing a doorway (door gap must remain)
- Leaving a window as opening (must be solid wall)

## Stage 04 — Trace
This is **outline tracing** (Illustrator's "Outline" mode), NOT skeletonization/centerline. Walls remain solid filled shapes; we capture their boundary.

**Library**: `esm-potrace-wasm` (current) or `imagetracerjs` (commercial-safe alt).

**Potrace options**:
- `turdsize: 20` (filter dust)
- `alphamax: 0.6` (favor corners over curves)
- `opticurve: false` (we want polylines, not Béziers)
- `opttolerance: 0.2`

**Orthogonalize**:
1. RDP simplify (`simplify-js`, epsilon ≈ 1.5px).
2. For each edge, if angle within ±`snapToleranceDeg` (default 12°) of an axis, lock x (vertical) or y (horizontal) to previous vertex.
3. Absorb edges shorter than `minSegmentLengthPx` (default 4) into neighbors.
4. Drop polygons with bbox area < ~25 px².

## Stage 05 — Walls
Each closed polygon → `THREE.Shape` → `ExtrudeGeometry(depth = wallHeight)`.

**Coordinate transform** (image → world):
```text
worldX = (px - imgWidth/2)  / pixelsPerMeter
worldZ = (py - imgHeight/2) / pixelsPerMeter   // image Y → world Z (Y-down to Z-forward)
worldY = 0 (floor) → wallHeightMeters (top)
```

## User controls

| Stage | Controls |
|---|---|
| 03 Clean | re-run cleanup button; optional manual paint/erase brush overlay |
| 04 Trace | snap_tolerance_deg, min_segment_length_px, turdsize sliders; per-path enable checkbox |
| 05 Extrude | wall_height_m input |

## Non-goals

- AI does not output geometry, paths, or coordinates.
- AI does not infer room semantics.
- Trace does not centerline / skeletonize walls.
- No 3D content is generated by AI — AI only paints raster pixels.
